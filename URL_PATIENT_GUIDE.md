# 📋 透過 URL 查看使用者資料指南

## 概述
本應用現已支援直接透過 URL 查看特定患者的資料。這讓您能夠：
- 🔗 直接分享患者資料連結
- 📌 收藏特定患者的個人資料頁面
- ⚡ 快速存取常用患者

## 📍 URL 格式

### 方式 1：直接訪問患者資料
```
http://localhost:3004/patients/{patientId}
```

### 方式 2：進入患者列表頁面
```
http://localhost:3004/patients
```
然後點擊「查看」按鈕或患者名稱

## 💡 使用範例

### 例 1：查看王大明的資料
```
http://localhost:3004/patients/1
```

### 例 2：查看陳老太的資料
```
http://localhost:3004/patients/2
```

### 例 3：查看全部患者列表
```
http://localhost:3004/patients
```

## ✨ 新增功能

### 自動導航
當您在患者列表頁面點擊患者時，URL 會自動更新為該患者的詳細資料頁面。

例如：
- 點擊「王大明」→ URL 變成 `/patients/1`
- 點擊「陳老太」→ URL 變成 `/patients/2`

### 返回列表
在患者詳細資料頁面的左上角，點擊「← 返回列表」按鈕即可回到患者列表。

## 🎯 功能列表

在患者詳細資料頁面，您可以：
- ✅ 查看基本資料（姓名、性別、生日、房間、聯絡人、電話）
- ✅ 查看日常用藥紀錄
- ✅ 查看病史
- ✅ **編輯資料**（點擊「編輯資料」按鈕）
- ✅ **新增/移除藥物**（在編輯模式下）
- ✅ **快速前往**相關頁面（每日健康、日常檢查）

## 🔄 支援的路由

| 路由 | 說明 |
|------|------|
| `/patients` | 患者列表頁面 |
| `/patients/:id` | 患者詳細資料頁面（ID 為患者編號） |

## 📱 技術資訊

### 實現細節
- 使用 React Router 的 `useParams` 鉤子處理 URL 參數
- 自動從選中行時更新 URL
- 患者資料從 API 或本機儲存中加載
- 支援患者資料的即時編輯和儲存

### 患者資料來源
- **API**：`/api/patients` (主要來源)
- **本機儲存**：localStorage（備援用）
- **Mock 數據**：本地模擬資料（初始化用）

## 📝 注意事項

1. **患者 ID**：使用數字 ID（1, 2, 3...），而不是字母 ID（p001, p002...）
2. **清除快取**：編輯後的資料會自動儲存至本機儲存
3. **無效 ID**：如果輸入不存在的患者 ID，會顯示患者列表 ⚠️
